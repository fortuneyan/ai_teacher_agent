## 二、备课阶段详细拆解

### 2.1 搜索课程标准

**真实场景**：
- 教师需要查找国家/地方课程标准
- 确定教学目标、核心素养要求
- 了解学段要求和内容边界

**信息来源**：
- 国家中小学智慧教育平台
- 各省市教育厅官网
- 学科教研网站
- 教材出版社资源

**关键决策**：
```
输入: 课程名称 + 教育阶段 + 学科领域
↓
搜索课标 → 解析目标 → 确定重点 → 提取关键词
↓
输出: 结构化课标数据（知识目标/能力目标/素养目标）
```

### 2.2 下载教材内容

**真实场景**：
- 获取官方教材电子版
- 分析章节结构和知识点
- 提取例题、习题、案例

**信息来源**：
- 出版社官方资源
- 学校教材库
- 开放教育资源

**关键决策**：
```
输入: 教材版本 + 出版社 + 年份 + 章节范围
↓
获取教材 → 内容解析 → 知识点提取 → 难度标注
↓
输出: 结构化教材内容（章节/知识点/例题/难度）
```

### 2.3 撰写教案

**真实场景**：
这是最核心的创造性工作，包括：

| 教案要素 | 具体内容 | AI辅助点 |
|---------|---------|---------|
| **教学目标** | 知识/能力/素养三维目标 | 基于课标自动生成 |
| **教学重难点** | 重点内容、难点分析 | 基于学情数据推荐 |
| **教学方法** | 讲授/讨论/探究/项目 | 基于内容类型匹配 |
| **教学过程** | 导入→新授→练习→小结→作业 | 时间分配优化 |
| **板书设计** | 核心知识结构图 | 自动生成可视化 |
| **教学反思** | 预设问题和应对 | 基于历史数据 |

**关键决策**：
```
输入: 课标 + 教材 + 学情 + 课时
↓
目标设计 → 过程规划 → 活动设计 → 资源准备
↓
输出: 完整教案文档
```

### 2.4 创建课件

**真实场景**：
- 将教案转化为可视化课件
- 设计互动环节
- 准备多媒体资源

**关键决策**：
```
输入: 教案 + 风格偏好 + 互动需求
↓
内容结构化 → 页面设计 → 动画配置 → 资源嵌入
↓
输出: PPT/交互课件
```

### 2.5 制定教学计划

**真实场景**：
- 规划整个学期的进度
- 平衡各章节时间
- 预留复习和考试

**关键决策**：
```
输入: 总课时 + 章节结构 + 节假日 + 考试安排
↓
时间分配 → 进度编排 → 里程碑设置 → 弹性预留
↓
输出: 教学进度表
```

## 三、授课阶段详细拆解

### 3.1 课堂导入（5-10分钟）

**真实场景**：
- 复习上节内容
- 引入新课主题
- 激发学习兴趣

**AI辅助点**：
- 根据学生掌握情况智能选择复习点
- 推荐情境导入素材
- 实时调整导入深度

### 3.2 知识讲解（20-30分钟）

**真实场景**：
- 讲解核心概念
- 演示例题
- 回应学生疑问

**AI辅助点**：
- 实时监测学生理解度（通过互动反馈）
- 自动推荐类比和例子
- 识别困惑点并调整讲解策略

### 3.3 互动问答（贯穿全程）

**真实场景**：
- 提问检查理解
- 学生提问解答
- 小组讨论引导

**AI辅助点**：
- 根据知识点自动生成问题
- 分析学生回答质量
- 推荐追问策略

### 3.4 课堂练习（10-15分钟）

**真实场景**：
- 随堂练习
- 即时反馈
- 针对性讲解

**AI辅助点**：
- 实时生成针对性练习题
- 自动批改和反馈
- 识别共性错误

### 3.5 总结与作业（5分钟）

**真实场景**：
- 总结本节重点
- 布置分层作业
- 预告下节内容

**AI辅助点**：
- 自动生成结构化总结
- 根据课堂表现推荐作业难度
- 个性化作业推荐

## 四、评估阶段详细拆解

### 4.1 出题

**真实场景**：
- 设计单元测试
- 期中/期末考试
- 日常小测

**出题维度**：
| 维度 | 说明 | AI辅助 |
|-----|------|--------|
| **难度分布** | 易:中:难 = 3:5:2 | 智能配比 |
| **题型组合** | 选择/填空/解答/应用 | 基于目标匹配 |
| **知识点覆盖** | 全面覆盖重点 | 自动检查覆盖度 |
| **区分度** | 能区分不同水平 | 基于历史数据优化 |

### 4.2 批改

**真实场景**：
- 客观题自动批改
- 主观题人工批改
- 作文/项目评价

**AI辅助点**：
- 客观题即时批改
- 主观题辅助评分（提供评分建议）
- 作文自动评价（结构/语言/内容）

### 4.3 学情分析

**真实场景**：
- 分析班级整体表现
- 识别薄弱知识点
- 发现学习困难学生

**分析维度**：
```
成绩数据
  ↓
├── 班级整体: 平均分/及格率/优秀率
├── 知识点维度: 各知识点掌握度
├── 学生维度: 个人成绩趋势
└── 题目维度: 难度/区分度分析
```

### 4.4 反馈生成

**真实场景**：
- 生成班级反馈报告
- 个性化学习建议
- 家长沟通内容

**AI辅助点**：
- 自动生成数据可视化报告
- 基于学情推荐改进策略
- 生成个性化学习路径

## 五、优化阶段详细拆解

### 5.1 课后反思

**真实场景**：
教师需要反思：
- 教学目标是否达成
- 教学方法是否有效
- 学生反馈如何
- 时间分配是否合理
- 意外情况处理

**反思框架**：
```
教学目标达成度: □ 完全达成 □ 基本达成 □ 未达成
学生参与度:    □ 高 □ 中 □ 低
时间控制:      □ 合理 □ 偏紧 □ 偏松
意外情况:      [记录]
改进方向:      [思考]
```

### 5.2 数据分析

**分析内容**：
- 课堂互动数据
- 练习正确率
- 学生提问分布
- 作业完成情况

### 5.3 教案优化

**优化方向**：
- 调整教学重难点
- 优化时间分配
- 改进教学活动
- 更新案例素材

### 5.4 课件更新

**更新内容**：
- 根据反思调整页面
- 更新数据和案例
- 优化互动设计

## 六、关键痛点与AI机会

### 6.1 教师痛点

| 痛点 | 描述 | AI解决方案 |
|-----|------|-----------|
| **备课耗时** | 撰写教案、找素材占大量时间 | 自动生成+智能推荐 |
| **个性化难** | 大班教学难以照顾个体差异 | 学情分析+分层推荐 |
| **反馈滞后** | 批改作业、分析学情耗时 | 自动批改+实时分析 |
| **经验传承** | 优秀教师经验难以复制 | 知识库+模式学习 |
| **持续优化** | 缺乏数据支持教学改进 | 数据分析+智能建议 |

### 6.2 AI教师Agent的核心价值

```
传统教师工作          AI辅助后
─────────────────────────────────────────
备课: 4小时    →     备课: 1小时（AI生成+人工调整）
批改: 2小时    →     批改: 0.5小时（AI辅助）
分析: 1小时    →     分析: 实时（自动完成）
反思: 主观     →     反思: 数据驱动
优化: 凭经验   →     优化: AI建议+效果预测
```

## 七、系统需求总结

### 7.1 功能需求

**备课模块**：
- [ ] 课标智能搜索与解析
- [ ] 教材内容获取与分析
- [ ] 教案自动生成与编辑
- [ ] 课件智能创建
- [ ] 教学计划自动编排

**授课模块**：
- [ ] 实时互动问答
- [ ] 课堂练习动态生成
- [ ] 学生理解度监测
- [ ] 智能板书生成

**评估模块**：
- [ ] 智能出题系统
- [ ] 自动批改（客观题）
- [ ] 主观题辅助评分
- [ ] 学情分析报告
- [ ] 个性化反馈生成

**优化模块**：
- [ ] 课后反思引导
- [ ] 教学数据分析
- [ ] 教案自动优化
- [ ] 课件智能更新

### 7.2 非功能需求

- **实时性**：课堂互动响应 < 1秒
- **个性化**：基于学生历史数据推荐
- **可解释**：AI决策可追溯
- **可控性**：教师可随时接管和调整
- **进化性**：从使用中学习和改进
